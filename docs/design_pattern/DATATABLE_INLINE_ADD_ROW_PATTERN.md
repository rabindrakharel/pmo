### Key Principles

1. **Cache IS the state** - No local copies, no useState for data
2. **Optimistic writes** - Update cache immediately, sync API in background
3. **Atomic rollback** - Capture previous state, restore on error
4. **No duplicate operations** - One place adds data, one place removes

### Cache Operations by State

| State | Operation | Cache Effect |
|-------|-----------|--------------|
| STABLE | Initial load | `useEntityInstanceData` populates cache from API |
| TEMP_ADDED | `handleAddRow` | `setQueryData` appends temp row |
| SAVING | `onMutate` | Captures previous state, NO new temp row |
| SAVED | `onSuccess` | Replace temp with real data, SKIP refetch |
| FAILED | `onError` | Filter out temp row from cache |
| CANCELLED | `handleCancel` | Filter out temp row from cache |

## 9. Reusable Framework

### Hook: `useInlineAddRow`

A reusable hook that encapsulates the entire inline add row pattern. Use this in any component that needs optimistic inline editing.

**Location**: `apps/web/src/db/cache/hooks/useInlineAddRow.ts`

### Basic Usage

```typescript
import {
  useInlineAddRow,
  createTempRow,
  shouldBlockNavigation,
  useOptimisticMutation,
} from '@/db/tanstack-index';

function MyEntityTable({ entityCode }: { entityCode: string }) {
  // Get mutation functions
  const { createEntity, updateEntity } = useOptimisticMutation(entityCode, {
    refetchOnSuccess: true,
  });

  // Use the reusable hook
  const {
    editingRow,
    editedData,
    isAddingRow,
    isSaving,
    handleAddRow,
    handleEditRow,
    handleFieldChange,
    handleSave,
    handleCancel,
    isRowEditing,
    isTempRow,
  } = useInlineAddRow({
    entityCode,
    createEntity,
    updateEntity,
    transformForApi: (data, original) => ({ ...original, ...data }),
    transformFromApi: (record) => ({ ...record }),
    debug: true,
  });

  // Create new row
  const onAddClick = () => {
    const newRow = createTempRow({ defaults: { status: 'draft' } });
    handleAddRow(newRow);
  };

  // Handle row click (block temp rows)
  const onRowClick = (row) => {
    if (shouldBlockNavigation(row.id)) return;
    navigate(`/${entityCode}/${row.id}`);
  };

  return (
    <Table
      data={data}
      editingRow={editingRow}
      onAddRow={onAddClick}
      onEdit={handleEditRow}
      onFieldChange={handleFieldChange}
      onSave={handleSave}
      onCancel={handleCancel}
      onRowClick={onRowClick}
    />
  );
}
```

### API Reference

#### `useInlineAddRow<T>(options)`

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `entityCode` | `string` | Yes | Entity code for cache key lookup |
| `createEntity` | `Function` | Yes | From `useOptimisticMutation` |
| `updateEntity` | `Function` | Yes | From `useOptimisticMutation` |
| `transformForApi` | `Function` | No | Transform editedData before API call |
| `transformFromApi` | `Function` | No | Transform record for editing |
| `onSaveSuccess` | `Function` | No | Callback after successful save |
| `onSaveError` | `Function` | No | Callback after save error |
| `onCancel` | `Function` | No | Callback after cancel |
| `debug` | `boolean` | No | Enable console logging |

#### Returns

| Property | Type | Description |
|----------|------|-------------|
| `editingRow` | `string \| null` | Currently editing row ID |
| `editedData` | `Partial<T>` | Accumulated field changes |
| `isAddingRow` | `boolean` | True if adding new row |
| `isSaving` | `boolean` | True if save in progress |
| `handleAddRow` | `(row: T) => void` | Add temp row to cache |
| `handleEditRow` | `(record: T) => void` | Enter edit mode for existing row |
| `handleFieldChange` | `(field, value) => void` | Update field in editedData |
| `handleSave` | `(record: T) => Promise<void>` | Save (create or update) |
| `handleCancel` | `() => void` | Cancel and cleanup |
| `resetEditState` | `() => void` | Clear all edit state |
| `isRowEditing` | `(id: string) => boolean` | Check if row is being edited |
| `isTempRow` | `(id: string) => boolean` | Check if row is temp |
| `getFieldValue` | `(field, original) => any` | Get current value |

#### `createTempRow<T>(options?)`

Creates a temp row with unique ID for adding to cache.

```typescript
const newRow = createTempRow<Project>({
  defaults: {
    dl__project_stage: 'planning',

  },
  generateName: () => 'New Project',

// Returns: { id: 'temp_1733288817235', name: 'New Project', _isNew: true, ... }
```

#### `shouldBlockNavigation(rowId)`

Check if navigation should be blocked (for temp rows).

```typescript
const handleRowClick = (row) => {
  if (shouldBlockNavigation(row.id)) {
    console.log('Cannot navigate to unsaved row');
    return;
  }
  navigate(`/${entityCode}/${row.id}`);
};
```

### Migration from Manual Implementation

If you have a component using the manual pattern, migrate to `useInlineAddRow`:

**Before (manual):**
```typescript
const [editingRow, setEditingRow] = useState(null);
const [editedData, setEditedData] = useState({});
const [isAddingRow, setIsAddingRow] = useState(false);

const handleAddRow = useCallback((newRow) => {
  const queryCache = queryClient.getQueryCache();
  // ... 20+ lines of cache manipulation
}, [queryClient, entityCode]);

const handleSaveInlineEdit = useCallback(async (record) => {
  // ... 30+ lines of save logic
}, [createEntity, updateEntity, editedData]);

const handleCancelInlineEdit = useCallback(() => {
  // ... 15+ lines of cancel logic
}, [queryClient, entityCode, editingRow]);
```

**After (reusable hook):**
```typescript
const {
  editingRow,
  editedData,
  isAddingRow,
  handleAddRow,
  handleSave,
  handleCancel,
} = useInlineAddRow({
  entityCode,
  createEntity,
  updateEntity,
});
```
